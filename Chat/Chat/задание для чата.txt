Действия клиента:
• отправка никнейма
{ type: "nickname", nickname: "..." }
• отправка текстового сообщения
{ type: "text", text: "something" }
• отправка файла
{ type: "file", filename: "...", binary: ... }
• отправка сообщения об ошибке
{ type: "error", message: "..." }

Действия сервера:
• отправка сообщения: дата/время, никнейм, сообщение
{ type: "text", "datetime": "...", nickname: "...", text: "..." }
• отправка файла: имя файла, бинарные данные
{ type: "file", filename: "...", binary: ... }
• отправка сообщения о завершении работы
{ type: "shutdown" }
• отправка сообщения об ошибке
{ type: "error", message: "..." }

==================================
{"type": "init", "nickname": "guest1"}
"type=init&nickname=guest1"
==================================


+ Должно отображаться время сообщения. Сервер при получении сообщения записывает время
(текущее время сервера в UTC). Отправляет сообщения всем пользователям с этим временем.
Клиенты уже сами преобразуют это время во время своего часового пояса

+ Пользователь при подключении должен ввести свой никнейм. Он передаётся на сервер, сервер
должен его запомнить на время его сессии. И все сообщения от этого пользователя
сопровождаются его никнеймом

Передача файлов.
+ Клиент должен иметь возможность каким-то образом указать файл для
передачи в чат. То есть, вводит путь к файлу, а затем он передаётся на сервер в бинарном
виде с указанием имени. После получения файла сервер должен отправить его всем участникам,
а затем удалить. Подключенные клиенты принимают все передаваемые файлы, записывают их на
диск и выводят сообщение в консоль, когда файл загружен

+ Добавить поддержку юникода

+ Сервер должен сообщить всем клиентам о завершении работы, когда оно происходит

+ Сообщать об ошибках при обмене данными

+ Проверить, что текст не смешивается
